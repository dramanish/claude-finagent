# Regulatory Language and Brand Compliance
 
## Purpose
This skill enforces KAELUM's mandatory terminology rules across all content, communications, and agent outputs. Apply to every output before it leaves any agent. Non-compliant language in investor materials, regulatory submissions, or public communications carries legal and reputational risk.
 
## Mandatory Brand Style
AI: always written as "Ai" (capital A, lowercase i) in all branded KAELUM contexts. This is a brand decision, not a typo. K.A.T.E.: always with full stops. KLM: never "token" standalone — use KLM, KLM unit, or commerce currency. KAELUM: always uppercase. Em-Dash/M-Dash: omit enrirely from all headings, sub-headings, body text and paragraphs.
 
## Prohibited Terms and Approved Replacements
crypto / cryptocurrency: use "commerce currency" or "Ai-governed currency." blockchain: use "closed-loop architecture." token (standalone): use "KLM," "KLM unit," or "commerce currency." wallet: use "account" or "KLM account." coin: use "KLM unit" or "KLM." minted / mined: use "issued," "allocated," or "credited." gas fee: omit entirely (not applicable). cashback: use "spending discount" or "merchant-side discount." interest / yield on holding: omit entirely (not applicable). deposit (implying savings return): omit entirely. multi-LLM: use "multi-instance Claude" or "specialised Claude instances." OpenAI / GPT / Gemini: omit entirely. Never reference competing AI systems. AI in branded contexts: use "Ai." em dash: never use. Replace with comma, semicolon, colon, or new sentence.
 
## Tagline
"Crypto is powered by Blockchain. KAELUM is powered by Ai." Use this verbatim when the brand tagline is required.
 
## Regulatory Framing Rules
Always lead with the problem before the solution in public content. Lead: "The digital commerce economy extracts value instead of creating it." Do not lead with KAELUM's features.
 
Never describe KAELUM as: a loyalty scheme, a rewards programme, a cryptocurrency alternative, a blockchain alternative, a savings product, an investment product.
 
Always describe KAELUM as: an Ai-governed, non-crypto, closed-loop digital commerce currency. A new category entirely.
 
KAELUM is live in "controlled onboarding phase" — never describe it as a full public launch.
 
## Investor Communication Rules
Always include in investor materials: SEIS/EIS eligibility, minimum £1.8M raise at £16.2M pre-money valuation, 10% equity. GTV targets: Year 1 £40M, Year 2 £250M, Year 3 £750M. Reference the EPI CEO and ECB President Lagarde validation (FT, February 2026) where the urgency context is appropriate.
 
## Content Approval Gate
All public-facing content (social posts, emails, press releases, web copy, articles) must pass regulatory language review before publication. The Content and Notification Agent routes all outputs to the admin approval queue. No autonomous publication is permitted.
